# parse-int-list

The v0 demo target for Yakcc.

## What this example demonstrates

`parse-int-list` is a program that parses a JSON array of integers — input
like `[1, 42, -7]` — and returns the integers as a typed array. It is the
first end-to-end target for the Yakcc assembly pipeline.

The program is composed entirely from hand-authored, content-addressed basic
blocks. No code is generated by AI at runtime. The assembler (`@yakcc/compile`)
retrieves the blocks from the registry by contract id and stitches them
together into a single runnable TypeScript file.

## Sub-blocks

The program is composed from the following basic blocks, each registered
independently under its own `ContractId`:

| Block | Contract | Description |
|---|---|---|
| `whitespace-skipper` | `cid:…` | Advances a cursor past leading whitespace characters. |
| `digit-recognizer` | `cid:…` | Returns true if a character is an ASCII decimal digit (0–9). |
| `integer-recognizer` | `cid:…` | Reads an optional leading `−` and one or more digits; returns a parsed integer. |
| `bracket-matcher` | `cid:…` | Asserts that the next non-whitespace character matches an expected bracket (`[` or `]`). |
| `comma-separator` | `cid:…` | Reads the comma between list elements, with optional surrounding whitespace. |
| `list-of-ints-parser` | `cid:…` | Composes the above blocks to parse a complete `[int, …]` expression. |

Each block satisfies a `ContractSpec` stored in the registry. The assembler
traverses the contract graph starting from `list-of-ints-parser` and emits the
full program in dependency order.

## Status

The running demo lands in **WI-005** (assembler wiring) and **WI-006**
(property-test runner). This folder describes the target; no runnable program
exists here yet.

To run the assembled program once WI-005 lands:

```
yakcc compile <list-of-ints-parser-contract-id> --out parse-int-list.ts
node --input-type=module < parse-int-list.ts
```
